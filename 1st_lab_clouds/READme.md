# Отчет по лабораторной работе №1: Знакомство с IaaS, PaaS, SaaS сервисами в облаке на примере Amazon Web Services (AWS). Создание сервисной модели.

## 1. Цель работы

Знакомство с облачными сервисами. Понимание уровней абстракции над инфраструктурой в облаке. Формирование понимания
типов потребления сервисов в сервисной-модели.

## 2. Исходные данные

Для анализа был предоставлен слепок биллинговых данных AWS в формате CSV, содержащий записи о потреблении различных
сервисов с полями:

Product Code — код продукта AWS (например, AmazonS3)
Usage Type — тип использования ресурса (например, %DataTransfer%)
lineItem/Operation — операция (зачастую пустая)
lineItem/LineItemDescription — описание строки счета

## 3. Ход работы

Мы разобрали счёт от AWS. Посмотрели, какие сервисы в нём есть (по полям Product Code и Usage Type) и разложили их по пяти уровням: от общей категории до конкретного использования: <br />
`IT Tower → Service Family → Service Type → Service Sub Type → Service Usage Type`. <br />
Мы распределили все платежи из биллинга по группам:

Storage — за хранение данных

Database — за базы данных

Networking — за сеть и передачу данных

Cloud Services — за готовые сервисы (типа машинного обучения)

Накладные расходы — например, налоги (Tax)

Мы выделили ключевые группы сервисов:
**Amazon S3** (Объектное хранилище)
Используется для хранения любых файлов: фото, видео, резервные копии, данные для аналитики. Надёжное, масштабируемое и безопасное хранилище с разными классами хранения (например, для частого доступа или для архивов).
Аналоги в РФ: Yandex Object Storage, SberCloud Object Storage, VK Cloud S3.

**Amazon QLDB** (Управляемые базы данных с журналом изменений)
База данных, которая сохраняет полную историю всех изменений. Подходит для систем, где важен контроль и прозрачность: финансы, логистика, документооборот.
Аналоги в РФ: Yandex Database (Yandex Cloud).

**Amazon Redshift** (Хранилище данных для аналитики)
Специализированное хранилище для работы с большими данными и сложной аналитики. Позволяет быстро выполнять SQL-запросы на огромных объёмах данных для отчётов и бизнес-аналитики.
Аналоги в РФ: Yandex DataProc, VK Cloud Big Data.

**Amazon VPC** (Виртуальная частная сеть)
Позволяет создать изолированную сеть в облаке AWS, где можно размещать виртуальные машины, базы данных и другие ресурсы. Контроль над IP-адресами, маршрутизацией и безопасностью.
Аналоги в РФ: Yandex Cloud Virtual Private Cloud, VK Cloud Network.

**Amazon SES и SNS** (Сервисы коммуникаций)

**SES** — для отправки email (рассылки, уведомления).

**SNS** — для push-уведомлений, SMS и обмена сообщениями между приложениями.
Аналоги в РФ: Yandex Cloud Message Queue, Mail.ru Notification Service.

## 4. Вывод
В ходе лабораторной работы мы построили модель использования сервисов AWS. Она помогает разбирать расходы по пяти уровням — от общих категорий до конкретных операций.
P.s Строки с Tax в lineItem/LineItemDescription отнесены к Overhead/Tax, остальные поля оставлены пустыми, так как налоги не детализируются по типам услуг.
